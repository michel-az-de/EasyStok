### E.1 Tela do Pedido — aba Pagamentos

```
┌──────────────────────────────────────────────────────────────┐
│ Pedido #1234 — Casa da Babá                          [x]    │
│ Cliente: Maria Silva — Apto 302                              │
│ ──────────────────────────────────────────────────────────── │
│ [Detalhes]  [Itens]  [Pagamentos •2]  [Histórico]            │
│ ──────────────────────────────────────────────────────────── │
│                                                              │
│  Total do pedido:        R$ 120,00                           │
│  Pago (confirmado):      R$  80,00                           │
│  ──────────────────────                                      │
│  Saldo a receber:        R$  40,00   [● Parcialmente pago]   │
│                                                              │
│  ┌──────────────────────────────────────────────────────┐    │
│  │ PIX  •  R$ 50,00  •  Confirmado                      │    │
│  │ 16/05 14:30  •  Thatiane  •  ref E12345abc           │    │
│  │                                          [Estornar]  │    │
│  ├──────────────────────────────────────────────────────┤    │
│  │ Dinheiro  •  R$ 30,00  •  Confirmado                 │    │
│  │ 16/05 14:35  •  Thatiane                             │    │
│  │                                          [Estornar]  │    │
│  └──────────────────────────────────────────────────────┘    │
│                                                              │
│  [+ Registrar novo pagamento]                                │
│                                                              │
│  ☐ Mostrar pagamentos estornados (0)                         │
└──────────────────────────────────────────────────────────────┘
```

**Fluxo 1 — Pagamento integral**:
1. Clica `[+ Registrar novo pagamento]`
2. Modal abre com valor pré-preenchido = saldo a receber (R$ 40,00)
3. Operador escolhe método via chips
4. Clica `[Confirmar pagamento]`
5. Modal fecha, lista atualiza, banner verde topo: "Pagamento registrado. Pedido totalmente pago."
6. Saldo a receber vira `R$ 0,00 [● Pago]`

**Fluxo 2 — Pagamento parcial**:
1. Igual ao 1, mas operador edita valor (ex: R$ 20,00 de R$ 40,00)
2. Confirma → lista cresce, saldo restante = R$ 20,00, badge `[● Parcialmente pago]`

**Fluxo 3 — Pagamento dividido (split)**:
1. Adiciona pagamento de R$ 20,00 (pix) — saldo: R$ 20,00
2. Repete: R$ 20,00 (dinheiro) — saldo: R$ 0,00, badge `[● Pago]`
3. Sem fluxo especial — apenas múltiplos pagamentos sequenciais.

**Estados visuais (badges)**:

| Estado | Cor (texto/fundo) | Hex |
|---|---|---|
| Aguardando pagamento | azul escuro/azul claro | `#1E40AF` / `#DBEAFE` (AAA 12.6:1) |
| Parcialmente pago | laranja/laranja claro | `#9A3412` / `#FFEDD5` (AAA 7.8:1) |
| Pago | verde/verde claro | `#166534` / `#DCFCE7` (AAA 9.3:1) |
| Cancelado com estorno pendente | vermelho/rosa claro | `#9F1239` / `#FFE4E6` (AAA 8.9:1) |
| Cancelado estornado | cinza/cinza claro | `#374151` / `#F3F4F6` (AAA 11.2:1) |

**Microcopy**:

- Error ao excedente: "O valor recebido (R$ 50) é maior que o saldo a receber (R$ 40). Ajuste o valor ou divida em dois pagamentos."

### E.4 Botão "Fechar caixa" — habilitação após X minutos

- **Tempo mínimo**: 10 minutos (config `Caixa:MinutosMinimosParaFechar` no
  `appsettings.json` — default 10, override por empresa via tabela
  `empresa_configs` se existir, senão hard-coded por enquanto).
- **Justificativa**: evita fechamento acidental logo após abertura. 10min é
  suficiente para usabilidade sem ser frustrante.
- **UX antes de 10min**: botão aparece com tooltip "Aguarde mais X min para
  fechar — abertura recente". Hover mostra contador.

### E.7 Acessibilidade WCAG 2.1 AA

**Cores e contraste** (todos AAA 7:1+):

| Elemento | Texto | Fundo | Contraste |
|---|---|---|---|
| Badge "Pago" | `#166534` | `#DCFCE7` | 9.3:1 |
| Badge "Parcialmente pago" | `#9A3412` | `#FFEDD5` | 7.8:1 |
| Badge "Aguardando" | `#1E40AF` | `#DBEAFE` | 12.6:1 |
| Badge "Estornado" | `#9F1239` | `#FFE4E6` | 8.9:1 |
| Botão "Fechar definitivamente" | `#FFFFFF` | `#B91C1C` | 5.9:1 |
| Cor de erro inline | `#991B1B` | `#FEF2F2` | 9.7:1 |

**`aria-live`**:
- Coluna "Diferença" no wizard passo 1: `aria-live="polite"` + `aria-atomic="true"`
- Banner topo (sucesso/erro pós-ação): `role="status"` + `aria-live="polite"`
- Toast de validação inline: `role="alert"` + `aria-live="assertive"`

**Foco visível**: outline `2px solid #1D4ED8` + `outline-offset: 2px` em
todos `:focus-visible`. Implementado em CSS global do PWA (já existe — confirmar).

**Tab order do wizard de fechamento (passo 1)**:
1. Inputs `contado` por linha de método (top→down)
2. Botão "Cancelar"
3. Botão "Próximo"

Skip-link "Pular para conteúdo principal" no topo da página caixa.

**Tradução de leitor de tela**:
- Badge: `<span aria-label="Status: pago integralmente">● Pago</span>`
- Divergência negativa: `<span aria-label="Diferença negativa de 1 real e 50 centavos">-R$ 1,50</span>`
